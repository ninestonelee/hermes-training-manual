goals

이 교재의 목표

이 교재는 헤르메스를 처음 접하는 사용자가 메신저 기반 AI 비서의 개념을 이해하고, 직접 연결하고, 반복 업무를 맡기는 데 필요한 흐름을 한 번에 익히도록 설계했습니다. 핵심 교육 방식은 쉽게 시작하기, 실제 업무 중심 실습, 말로 지시하기, 안전을 먼저 다루기입니다.

  • 대상: 코딩을 깊게 모르는 일반 사용자, 교육 운영자, 업무 자동화 입문자
  • 학습 결과: 구조 이해, 메신저 연결, 첫 대화, 알림/브리핑 자동화, 정보 수집 실습
  • 운영 원칙: 편리함보다 안전을 먼저 정하고 작은 자동화부터 확장

benchmark

벤치마킹 요약

원자료에서 확인한 교육 패턴을 헤르메스용 교재 설계로 변환했습니다. 원문을 복제하지 않고 구조, 난이도, 실습 흐름만 새롭게 적용했습니다.

관찰한 패턴 헤르메스 교재 적용 방식
아키텍처를 그림으로 먼저 설명 채널, 게이트웨이, 에이전트, 도구의 역할을 쉬운 비유로 정리
텔레그램을 쉬운 입문 채널로 사용 텔레그램을 기본 예제로 두고 Discord, Slack, WhatsApp은 확장 채널로 안내
명령어보다 자연어 지시를 강조 복사해서 바로 쓸 수 있는 “이렇게 말하세요” 문장 제공
보안 주의를 초반에 배치 토큰, 앱 비밀번호, 파일 권한, 웹 접근 범위를 1장부터 다룸
크론 잡과 이메일 브리핑을 실용 예제로 제시 알림, 날씨, 일정, 이메일, 웹 요약을 단계별 프로젝트로 구성

PART 01

처음 만나는 헤르메스

Chapter 01. 헤르메스는 무엇인가

헤르메스는 채팅창 안에만 머무르는 챗봇이 아니라, 사용자의 컴퓨터와 연결된 AI 작업 비서입니다. 사용자는 익숙한 메신저에서 말을 걸고, 헤르메스는 게이트웨이를 통해 요청을 받은 뒤 필요한 도구를 사용해 일을 처리합니다.

메신저 채널
게이트웨이
AI 에이전트
도구/스킬

Chapter 02. 시작하기 전에 준비할 것

  • 항상 켜둘 컴퓨터 또는 서버를 준비한다.
  • 터미널에서 현재 위치 확인, 폴더 이동, 목록 보기 정도는 연습한다.
  • AI 모델 API 키 또는 구독 계정을 준비한다.
  • 연결할 메신저 계정을 정한다. 입문용으로는 텔레그램을 권장한다.
  • 실습용 폴더를 따로 만들고 중요한 개인 파일과 분리한다.
보안 원칙

API 키와 봇 토큰은 암호와 같습니다. 채팅방, 깃허브, 문서에 그대로 붙여 넣지 않고, 처음에는 헤르메스가 접근할 수 있는 폴더와 웹사이트를 좁게 제한하세요.

PART 02

헤르메스 설치와 첫 대화

Chapter 03. 게이트웨이 실행하기

게이트웨이는 헤르메스의 출입문입니다. 사용자의 메시지, 예약 작업, 외부 도구 요청이 모두 이곳을 통과합니다. 

hermes gateway run --port 18789
[Hermes Gateway] Starting...
[Agent] Ready
[Gateway] Listening on port 18789

Chapter 04. 텔레그램 봇 연결하기

  1. 텔레그램에서 BotFather를 검색하고 대화를 시작합니다.
  2. /newbot을 입력하고 봇 이름과 사용자명을 정합니다.
  3. 발급된 토큰을 안전한 비밀 저장소에 보관합니다.
  4. 헤르메스 설정에 토큰을 등록합니다.
  5. 봇 대화창에서 짧게 인사해 응답을 확인합니다.

안녕, 나는 준이고 너는 내 업무 비서 헤르메스야. 앞으로 나를 도와 일정 정리, 이메일 요약, 자료 조사, 알림 설정을 해줘. 먼저 네가 할 수 있는 일을 짧게 소개해줘.

PART 03

말로 시키는 자동화

Chapter 05. 20분 뒤 알림 만들기

가장 작은 자동화는 일회성 알림입니다. 사용자는 명령어를 외울 필요 없이 자연어로 말하면 됩니다.

20분 뒤에 ‘회의실 예약 확인하기’라고 알려줘.

구분 언제 쓰나 예시
일회성 알림 한 번만 알려주면 되는 일 내일 오전 10시에 병원 예약 확인
반복 브리핑 매일 또는 매주 반복되는 일 평일 오전 9시에 오늘 일정 요약
상태 감시 문제가 생겼을 때만 알고 싶은 일 5분마다 서버 상태 확인

Chapter 06. 아침 브리핑 만들기

헤르메스가 비서처럼 느껴지는 순간은 사용자가 먼저 묻지 않아도 필요한 정보를 챙겨줄 때입니다. 아침 브리핑은 날씨, 일정, 할 일, 중요한 이메일을 한 번에 묶는 대표 실습입니다.

매일 평일 오전 8시에 오늘 날씨, 캘린더 일정, 중요한 이메일, 오늘 해야 할 일을 5줄 이내로 브리핑해줘.

Chapter 07. 이메일 연결과 앱 비밀번호

이메일 자동화는 강력하지만 민감합니다. 개인 계정 비밀번호를 그대로 쓰지 말고, 가능하면 앱 비밀번호나 별도 토큰을 사용하세요. 

hermes skill install email
hermes secret set gmail_app_password
hermes email connect --account [REDACTED_EMAIL]

PART 04

정보 수집과 웹 요약

헤르메스는 웹페이지를 열어 내용을 읽고 핵심을 요약할 수 있습니다. 단, 불필요하게 자주 접속하거나 대량으로 긁어오는 방식은 피해야 합니다.

웹페이지 내용을 읽고 핵심만 요약해줘. 제목, 핵심 이슈 5개, 내가 확인해야 할 행동 3개로 정리해줘: https://example.com

  • 같은 페이지를 초 단위로 반복 새로고침하지 않습니다.
  • 한 번에 수백 페이지를 수집하지 않습니다.
  • 서비스 약관과 robots.txt를 확인합니다.
  • 출처와 날짜를 요약에 함께 남깁니다.
