로컬 설치 절차

Hermes Agent 설치와 첫 실행

Windows, Mac, Linux에서 Hermes를 설치하고 첫 대화를 시작하기 위한 절차형 매뉴얼입니다. 기존 강의 핵심은 유지하되 공식 문서와 충돌하는 비용·권한 표현은 안전하게 정리했습니다.

1. 사전조건

Hermes 설치 프로그램은 필요한 런타임 대부분을 자동으로 준비합니다. 사용자는 터미널, 브라우저 로그인, 모델 provider 선택지만 준비하면 됩니다.

  • Windows: PowerShell 또는 Windows Terminal. 네이티브 설치는 early beta이며, 더 안정적인 경로가 필요하면 WSL2를 사용합니다.
  • Mac / Linux / WSL2: 기본 터미널과 네트워크 연결.
  • 브라우저 로그인: Nous Portal, OpenAI, Anthropic, Google 등 사용할 provider 계정.
  • 비밀키 관리 원칙: API 키는 채팅창이나 문서에 쓰지 않습니다. 설정 마법사나 전용 config 명령을 사용합니다.
i

관리자 권한은 기본값이 아닙니다. Windows 네이티브 설치는 사용자 영역에 설치되며 공식 문서 기준으로 관리자 권한이 필요하지 않습니다. 권한 오류가 날 때만 별도로 점검하세요.

2. 설치

운영체제에 맞는 한 줄 설치 명령을 실행합니다. Windows native와 WSL2 설치는 데이터 위치가 다르므로 서로 충돌하지 않습니다.

Windows native: PowerShell

  1. PowerShell을 엽니다. 시작 메뉴에서 PowerShell 또는 Windows Terminal을 실행합니다.
  2. 설치 명령을 실행합니다. 아래 명령을 붙여넣고 Enter를 누릅니다.
  3. 새 터미널을 엽니다. 설치 후 PATH 반영을 위해 PowerShell 창을 새로 엽니다.
PowerShell
iex (irm https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.ps1)

Mac, Linux, WSL2

  1. 터미널을 엽니다. macOS Terminal, zsh, bash, Linux shell 중 하나를 사용합니다.
  2. 설치 명령을 실행합니다. 아래 명령은 git 기반 설치를 수행합니다.
  3. 셸을 다시 로드합니다. 설치 완료 후 새 터미널을 열거나 셸 설정을 다시 읽습니다.
bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

3. 설정

설치가 끝나면 설정 마법사 또는 개별 명령으로 provider, 모델, 도구, 메시징을 설정합니다. 처음에는 간단 설정을 완료한 뒤 필요할 때만 세부 설정으로 넘어갑니다.

  1. 전체 설정 마법사를 실행합니다. 새 터미널에서 hermes setup을 실행하거나 설치 직후 나타나는 안내를 따릅니다.
  2. Provider를 선택합니다. 비용과 인증 방식을 확인한 뒤 Nous Portal, OpenAI, Anthropic, Google, 로컬 모델 중 하나를 선택합니다.
  3. 터미널 백엔드를 고릅니다. 입문자는 Local을 선택합니다. VPS나 SSH 백엔드는 운영·확장 문서에서 다룹니다.
  4. 위험 권한을 제한합니다. sudo support 또는 관리자 권한 자동 사용은 필요할 때만 켭니다.
  5. 메시징은 나중에 붙입니다. Telegram, Discord는 설치 검증 후 설정하면 장애 원인을 분리하기 쉽습니다.
설정 명령
hermes setup
hermes model
hermes tools
hermes gateway setup
!

무료·무제한 표현을 그대로 믿지 마세요. Provider의 무료 플랜, 구독 포함량, OAuth 사용 가능 여부는 자주 바뀝니다. 실제 과금 조건은 설정 화면과 공식 provider 문서에서 확인한 뒤 선택합니다.

4. 실행

설정이 끝나면 대화형 CLI를 시작합니다. 첫 실행에서는 간단한 자기소개, 작업 디렉터리 확인, 기억 파일 위치 확인 정도만 진행합니다.

첫 실행
hermes

첫 작업 예시

  1. 정체성을 부여합니다. 원하는 이름, 역할, 말투를 알려주고 SOUL 파일에 반영하도록 요청합니다.
  2. 환경을 알려줍니다. 자주 쓰는 폴더, 프로젝트, 도구, 선호를 알려주면 USER와 MEMORY 파일에 정리됩니다.
  3. 작은 반복 작업을 맡깁니다. 파일 정리, 뉴스 요약, 점검 목록 작성처럼 위험이 낮은 작업부터 시작합니다.
  4. 웹 대시보드를 확인합니다. 스킬, 세션, 작업 상태는 브라우저에서 http://localhost:9119로 확인합니다.
i

설치는 시작점입니다. Hermes의 핵심 가치는 기억, 스킬, 크론, 메시징이 쌓이면서 “반복 작업을 맡기는 방식”이 안정되는 데 있습니다.

5. 검증

설치 직후에는 기능을 넓히기보다 기본 동작을 좁게 검증합니다.

  1. 명령 인식 확인hermes --help 또는 hermes doctor가 실행되는지 확인합니다.
  2. 대화 확인hermes를 실행해 짧은 질문에 응답하는지 확인합니다.
  3. Provider 확인hermes model에서 현재 provider와 모델을 확인합니다.
  4. 대시보드 확인http://localhost:9119를 열어 화면이 표시되는지 확인합니다.
  5. 경로 확인Windows는 사용자 프로필과 %LOCALAPPDATA%\hermes, Mac/Linux는 ~/.hermes에 데이터가 쌓이는지 확인합니다.
진단 명령
hermes doctor
hermes model

6. 장애 대응

증상대응
hermes 명령을 찾지 못함터미널을 새로 열고 PATH를 다시 반영합니다. Mac/Linux는 셸 설정 파일을 다시 읽습니다.
Windows 설치가 불안정함네이티브가 early beta인 점을 감안해 WSL2 경로로 재설치하는 것을 검토합니다.
로그인 또는 캡차 실패기본 브라우저를 바꾸거나 provider 인증을 다시 실행합니다.
API 키 또는 인증 오류hermes model로 provider를 다시 선택하고, 필요한 키는 채팅이 아니라 설정 경로로 입력합니다.
한글 출력이 깨짐Windows Terminal을 사용하고, 인코딩 관련 환경 변수를 공식 문서 기준으로 점검합니다.
대시보드 일부 기능이 안 보임Windows native는 embedded terminal pane 등 일부 기능 차이가 있을 수 있습니다. CLI와 기본 대시보드부터 검증합니다.
i

막힌 지점에서는 에러 메시지를 그대로 복사해 진단 요청을 하세요. 단, 토큰·API 키·개인정보는 먼저 제거합니다.

7. 복구

복구는 “설치 파일”과 “개인 데이터”를 분리해서 생각합니다. Hermes 재설치는 쉬우며, 중요한 것은 기억, 스킬, 설정, 비밀키를 안전하게 다루는 것입니다.

  1. 설정만 다시 하기hermes setup 또는 hermes model로 provider와 모델을 다시 잡습니다.
  2. 인증 다시 하기provider 계정의 세션이 만료되면 인증 명령을 다시 실행합니다.
  3. 설치 복구PATH나 portable Git이 깨졌다면 공식 설치 명령을 다시 실행합니다. Windows portable Git 문제는 사용자 영역 Git 폴더를 정리한 뒤 재실행할 수 있습니다.
  4. 비밀키 유출 대응채팅이나 문서에 키를 붙여넣었다면 즉시 발급처에서 폐기하고 새 키를 발급합니다.
  5. 백업에서 복원스킬과 메모리를 GitHub 등 비공개 저장소에 백업해두었다면 새 설치 후 해당 파일을 다시 가져옵니다.

8. 수동 검증 체크리스트

  • 키보드만으로 skip link, 상단 내비게이션, 목차, 복사 버튼에 접근했다.
  • 모바일 폭에서 목차가 접히고, 본문과 코드 블록이 가로로 깨지지 않는다.
  • 모든 외부 링크가 새 탭으로 열리고 rel="noopener noreferrer"를 포함한다.
  • 브라우저 저장소 API에 의존하지 않아 sandbox iframe에서도 동작한다.
  • 설치 명령, 실행 명령, 진단 명령을 복사해도 임시값이나 비밀키가 포함되지 않는다.
  • Windows native와 WSL2의 차이, early beta 경고, 관리자 권한 안내가 명확하다.
  • 깨진 내부 앵커와 죽은 로컬 링크가 없다.
