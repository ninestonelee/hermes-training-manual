텔레그램으로 부리기

24시간 켜둔 서버에 깔린 헤르메스(Hermes Agent)를, 봇파더부터 그룹·토픽까지 순서대로 따라만 하면 텔레그램에서 바로 명령할 수 있게 됩니다. 각 단계를 끝낼 때마다 체크하면 진행률이 차오릅니다.

🖥️ 24시간 서버(내 PC 또는 VPS)에 헤르메스 설치 완료 📨 텔레그램 계정 🔑 LLM 키(OpenAI·OpenRouter 등)

시작해 볼까요? 7단계 중 0단계 완료
// 설치 & 연결
1✓
봇파더에서 봇 만들기 @BotFather

텔레그램 봇은 전부 @BotFather에서 시작합니다. 새 봇을 만들고 봇 토큰을 받아두는 단계예요.

  1. 텔레그램에서 @BotFather 검색 후 대화 시작
  2. /newbot 입력
  3. 표시 이름 입력 — 자유롭게 (예: 헤매봇)
  4. 사용자명 입력 — 전 세계에서 고유하고 반드시 bot으로 끝나야 함 (예: zero_haemae_bot)
  5. 봇파더가 주는 봇 토큰 복사 (아래 형태)
bot token
123456789:ABCdefGHIjklMNOpqrSTUvwxYZ
⚠️
토큰은 비밀번호입니다. 가진 사람은 누구나 봇을 조종할 수 있어요. 유출되면 봇파더에서 /revoke로 즉시 재발급하세요.
2✓
봇 메뉴 설정 + 프라이버시 끄기 @BotFather

슬래시 메뉴를 등록하고, 그룹에서 안 막히도록 프라이버시 모드를 손봅니다. 영상에서 "그룹인데 답을 안 한다"고 헤맸던 바로 그 부분이에요.

  1. @BotFather에 /setcommands → 내 봇 선택 → 아래 목록 붙여넣기
command menu
new - 새 대화 시작
model - AI 모델 보기/변경
status - 세션 정보
help - 도움말
sethome - 이 채팅을 홈 채널로 지정
  1. /mybots → 내 봇 → Bot Settings → Group Privacy → Turn off
🔁
설정을 바꾼 뒤에는 봇을 그룹에서 내보냈다가 다시 초대해야 합니다. 텔레그램은 봇이 들어온 시점의 프라이버시 상태를 캐싱하기 때문이에요.
🔎
더 쉬운 대안: 프라이버시를 끄는 대신 봇을 그룹 관리자(admin)로 올리면, 설정과 무관하게 모든 메시지를 받습니다. 영상의 결말도 "관리자 등록이 빠져 있었다"였죠.
3✓
내 User ID 알아내기 @userinfobot

헤르메스는 @아이디가 아니라 숫자 User ID로 권한을 관리합니다. 이 숫자를 미리 확보해 두세요.

  1. 텔레그램에서 @userinfobot 에게 아무 메시지나 전송
  2. 즉시 답해주는 숫자 ID(예: 123456789) 복사 — 대안: @get_id_bot
4✓
헤르메스에 텔레그램 연결 서버 터미널

서버에서 게이트웨이 설정 마법사를 돌립니다. 게이트웨이는 헤르메스 ↔ 텔레그램을 잇는 중간 다리예요.

방법 A — 대화형 (권장)

terminal
$ hermes gateway setup   # Telegram 선택 → 토큰 붙여넣기 → User ID 입력

방법 B — 수동 · ~/.hermes/.env 파일에 추가

~/.hermes/.env
TELEGRAM_BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrSTUvwxYZ
TELEGRAM_ALLOWED_USERS=123456789   # 여러 명이면 쉼표로
💡
이 단계에서 User ID를 그냥 넘어가도 됩니다. 나중에 텔레그램에서 "지금부터 나만 메시지 보낼 수 있게 해줘"라고 말로 시켜도 돼요. (단, 보안상 꼭 해두세요.)
5✓
게이트웨이 실행 & 24시간 가동 서버 터미널

먼저 테스트로 띄워 응답을 확인하고, 그 다음 재부팅에도 살아남는 서비스로 등록합니다.

① 테스트 실행 — 봇에 메시지 보내 응답 확인 후 Ctrl+C

terminal
$ hermes gateway
[Gateway] Telegram adapter connected ✓

② 서비스로 설치 & 관리

terminal
$ hermes gateway install              # 사용자 서비스 설치
$ sudo hermes gateway install --system # 부팅 시 자동 시작(Linux)
$ hermes gateway start                # 시작 (stop/status/restart)
$ sudo loginctl enable-linger $USER    # SSH 끊겨도 계속 가동
💡
설정을 바꿔 재시작이 필요할 땐 직접 안 해도 됩니다. 봇에 /update를 보내거나 "너 스스로 게이트웨이 재시작해"라고 말하면 알아서 처리해요.
6✓
권한 — 나만 / 우리 팀만 쓰게 서버 터미널

허용목록을 미설정하면 게이트웨이는 기본적으로 모든 사용자를 거부합니다. 본인 ID를 꼭 넣으세요.

~/.hermes/.env
TELEGRAM_ALLOWED_USERS=123456789,987654321  # 변경 후 gateway 재시작
🔎
팀이라면 DM 페어링이 편합니다. 팀원이 봇에 DM하면 일회용 코드(예: XKGH5N7P)가 발급되고, 소유자가 서버에서 hermes pairing approve telegram XKGH5N7P로 승인하면 즉시 사용 가능 — 게이트웨이 재시작도 불필요. (코드는 1시간 후 만료)
7✓
그룹 + 토픽으로 주제 분리 Telegram 앱

한 봇으로 개발 / 일상 / 재테크처럼 맥락이 완전히 분리된 대화를 동시에. 헤르메스가 여러 작업을 병렬로 처리하게 만드는 핵심 활용법입니다.

  1. 텔레그램 → New Group으로 그룹 생성(예: "헤매봇 비서실") → 봇을 멤버로 추가
  2. 그룹 → Manage Group → Topics 토글 켜기
  3. 그룹 → Administrators에 봇을 관리자로 추가 → 저장
  4. 상단 토픽 영역 → Create Topic으로 재테크 일상 개발 생성
  5. 토픽 안에서 봇 멘션으로 호출 → 토픽마다 다른 주제로 대화
in topic chat
# '일상' 토픽에서
@zero_haemae_bot 오늘 저녁 뭐 먹지?
⚠️
그룹 진입 직후 바로 입력하면 '일반(General)' 영역으로 갑니다. 항상 올바른 토픽을 선택했는지 확인 후 메시지를 남기세요. 그룹에서도 한 번 /sethome 실행을 권장합니다.
🔎
그룹 없이도 가능: 봇 DM에서 /topic을 켜면 1:1 대화 안에서도 토픽(병렬 세션)을 만들 수 있어요. (Bot API 9.4 신기능 — 사전에 BotFather Threads Settings 활성화 필요)
// 알아두면 좋은 사용법

작업 중에 메시지 보내기 — 3가지 모드

헤르메스가 작업 중일 때 추가 메시지를 어떻게 처리할지 결정합니다. 봇 이름 옆 "입력 중(typing)" 표시가 떠 있으면 작업 중이라는 신호예요.

STEER · 기본

말 끼어들기

그냥 메시지 입력. 하던 작업을 즉시 멈추고 새 지시에 집중. 오타·잘못된 지시를 바로잡을 때.

QUEUE · 큐

이어서 하기

지금 작업을 완료한 뒤 이 작업을 순서대로 이어서 수행.

BACKGROUND

동시에 하기

/background 로 별도 인스턴스에서 동시 실행. 과정은 안 보이고 결과만 도착.

// 슬래시 명령어 치트시트
/new새 대화(세션) 시작
/model인라인 버튼으로 제공자·모델 선택 (/model 이름으로 즉시 변경)
/status현재 세션 정보
/whoami내 권한 등급과 실행 가능한 명령 확인
/sethome이 채팅/토픽을 홈 채널로 지정 (예약 작업 결과가 도착)
/background동시 백그라운드 작업 실행
/topicDM 안에서 멀티세션(토픽) 모드 켜기
/update최신 버전 업데이트 후 자동 재시작
/stop현재 작업 중지

1
텔레그램이 메시지를 전달하는가

봇파더 프라이버시 끄기 또는 봇을 그룹 관리자로 승격

2
프라이버시 변경 후 재초대했는가

봇을 내보냈다 다시 초대 (캐싱 때문)

3
발신자가 허용목록에 있는가

TELEGRAM_ALLOWED_USERS / TELEGRAM_GROUP_ALLOWED_CHATS 확인

4
멘션 필터

require_mention: true면 멘션·답장·슬래시명령일 때만 반응

5
도구 호출이 중간에 끊김

"이터레이션 최대 횟수를 90회로 늘려줘"라고 지시 (기본 30이면 잦은 끊김)

// 트러블슈팅 — "텔레그램 웹 로그인이 무한 대기에 걸릴 때"

web.telegram.org에서 전화번호 로그인·QR 로그인이 자주 무한 대기에 걸려 접속이 막힐 때가 있어요. 이때는 패스키(Passkey) 로그인이 빠르고 안정적입니다. 단, 패스키는 모바일 앱에서 먼저 만들어야 웹에서 쓸 수 있어요.

① 모바일 앱에서 패스키 만들기

1
텔레그램 앱 → 설정 → 개인정보 및 보안 → 패스키

앱을 최신 버전으로 업데이트해 두세요 (Android/iOS 안정 버전 지원)

2
"패스키 생성"

페이스ID·터치ID·지문 또는 기기 PIN으로 인증

3
저장 완료

iCloud 키체인(Apple) 또는 Google 비밀번호 관리자(Android)에 동기화

② 웹에서 패스키로 로그인

1
web.telegram.org 접속

사용자 이름(또는 번호) 입력

2
"패스키로 로그인" 선택

SMS 코드·QR 무한 대기 대신 패스키 사용

3
생체 인증/PIN → 완료

SMS 가로채기 위험도 함께 차단

💡 패스키를 켜도 계정은 활성 전화번호에 등록돼 있어야 합니다. · 참고: 텔레그램 공식 패스키 안내
