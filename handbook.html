// 5. cron 스케줄링 — "Morning Brief" 실제 구현
7✓
전제: gateway 데몬 띄우기 필수

영상이 안 알려준 핵심. cron은 gateway 데몬이 60초마다 스케줄러를 tick 하며 실행합니다. gateway가 없으면 cron이 안 돕니다.

terminal
$ hermes gateway install            # 사용자 서비스 (로그인 시 자동)
$ sudo hermes gateway install --system # Linux 서버: 부팅 시 시스템 서비스
$ hermes gateway                    # 또는 포그라운드 직접 실행
8✓
cron 작업 생성 (자연어 / CLI) cron

방법 A: 자연어 — gateway만 떠 있으면 그냥 말하면 내부 cronjob 툴로 생성됩니다.

자연어
Every morning at 8am, send me a Morning Brief with:
1) today's weather, 2) a motivational quote,
3) my meetings today. Deliver to Telegram.

방법 B: CLI — 정확·재현 가능(권장).

terminal
$ hermes cron create \
    --schedule "0 8 * * *" \
    --prompt "Morning Brief: weather in Seoul, one quote, my meetings." \
    --deliver telegram
관리 명령용도
hermes cron list작업 목록
hermes cron status스케줄러 동작 여부 확인
hermes cron edit <id> --schedule "every 4h"스케줄 변경
hermes cron pause / resume <id>일시정지 / 재개
hermes cron run <id>다음 tick에 강제 실행
hermes cron remove <id>삭제
hermes cron logs <id>실행 이력·에러 확인
9✓
"6am Dreaming Sequence"를 실제로 — cron 2개 분리 advanced

영상의 "브리핑 전 6시에 미리 dream"은 cron 작업 2개로 구현합니다.

terminal — 6am dream + 8am brief
# 6am: 모든 맥락 기반 추천 3개 + 절대 할 것 1개 → 노트 저장
$ hermes cron create --schedule "0 6 * * *" \
    --prompt "Based on all history/meetings/projects, write 3
    recommendations + 1 non-negotiable. Save to today's note." \
    --deliver none

# 8am: 위 결과 + 날씨/명언/일정 묶어 발송
$ hermes cron create --schedule "0 8 * * *" \
    --prompt "Compile Morning Brief: weather, quote, meetings,
    and 3 recs + non-negotiable from the 6am note." \
    --deliver telegram
cron 제약(공식): cron 세션은 또 다른 cron을 못 만듭니다(runaway 방지). 최종 응답은 자동 전달 — 프롬프트에 send_message 불필요. --deliver all로 모든 채널 일괄 발송. 무인 실행엔 hermes setup --portal(OAuth 자동 갱신)이 안전.
파이프라인 매핑: 당신이 구상한 "event-driven hooks vs cron 스케줄링"에서 Hermes cron은 능동(proactive) 축, 메시징 게이트웨이가 반응(reactive) 축을 담당. 둘 결합 시 "hooks + cron + Obsidian 누적" 아키텍처가 상당 부분 재현됩니다.

// 9. OpenClaw / Anti-gravity 비교 & 마이그레이션
12✓
OpenClaw → Hermes 자동 마이그레이션 migrate

영상 결론은 "여러 비서 돌리지 말고 하나에 집중". hermes setup이 ~/.openclaw를 자동 감지해 설정·메모리·스킬·API 키 이전을 먼저 제안합니다.

terminal
$ hermes claw migrate              # 대화형 (전체 프리셋)
$ hermes claw migrate --dry-run    # 미리보기 — 먼저 이것부터!
$ hermes claw migrate --preset user-data # 비밀값 제외 이전
$ hermes claw migrate --overwrite  # 충돌 시 덮어쓰기
검증 루프: --dry-run으로 무엇이 옮겨질지 반드시 먼저 확인한 뒤 실제 실행.

  • 영상 권장: 본인 컴퓨터에서 24/7 실행이 가장 쉬움. VPS는 터널링·보안 미흡 시 공격 위험.
  • 파일 격리 원하면 Docker 컨테이너로 Hermes만 샌드박스 실행.
  • 메모리·Obsidian이 로컬에 있으면 로컬 실행이 연결 단순 — 당신의 Termux/Obsidian 구조와 부합.
추정: 안드로이드 상시 cron 구동은 Termux wake-lock 등 별도 설정과 발열·배터리 고려가 필요합니다.

// 11. /goal 제대로 쓰기 — "Super Goal" 공식 대안

사람↔AI 핸드셰이크를 기본 /goal로 재현

"Super Goal"은 발표자 자작 유료 스킬. 핵심 아이디어(작업을 잘게 쪼개 일부는 AI, 일부는 사람이 수행)는 기본 /goal + 명확한 프롬프트로 재현 가능합니다.

영상 교훈: 목표는 반드시 구체적·검증가능하게. ❌ "내 비즈니스를 키워줘"(출력 정의 없음→끝없이 돔) / ✅ "이 기준 3개를 충족하는 문서" "이런 모양의 썸네일".
Goal 프롬프트 (핸드셰이크 내장 · 복붙)
> /goal Launch my [프로젝트] to [측정가능 목표, 예: 500 signups].

Break this into bite-sized tasks. Label each [AI] or [ME]:
- [AI] tasks: you do autonomously.
- [ME] tasks: list clearly what only I can do (e.g. record videos).

Define a concrete "done" criterion so you know when to stop.
After each milestone, show me the checklist status first.
