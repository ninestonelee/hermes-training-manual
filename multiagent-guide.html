멀티 에이전트 절차

Discord 작업실과 리뷰 에이전트 운영

Discord 채널과 스레드를 작업 단위로 사용하고, 구현 담당과 리뷰 담당 에이전트를 분리해 협업시키는 운영 매뉴얼입니다.

대상메시징 gateway를 운영할 사용자
환경Discord, Hermes gateway, 프로필
결과채널·스레드 기반 협업 운영

1. 사전조건

멀티 에이전트 운영은 설치보다 운영 규칙이 중요합니다. 처음부터 많은 봇을 만들기보다 한 채널, 한 구현 봇, 한 리뷰 봇으로 검증합니다.

  • Hermes 기본 운영 완료: 로컬 또는 VPS에서 hermes와 gateway가 정상 동작해야 합니다.
  • Discord 서버 권한: 애플리케이션을 만들고 봇을 초대할 수 있는 권한이 필요합니다.
  • 사용자 ID 확인: Discord 개발자 모드를 켜고 본인 User ID를 복사할 수 있어야 합니다.
  • 채널 규칙: 채널은 프로젝트, 스레드는 작업 단위로 사용합니다.
  • 권한 제한: 허용 사용자, 역할, 채널을 제한하고 봇 토큰은 절대 공개하지 않습니다.
i

Telegram은 빠른 리모컨에 가깝고 Discord는 프로젝트 작업실에 가깝습니다. 여러 작업을 병렬로 굴릴수록 채널과 스레드 분리가 중요해집니다.

2. 설치

Discord 연동은 Discord 애플리케이션 생성, 봇 초대, Hermes gateway 연결 순서로 진행합니다.

  1. Discord 애플리케이션을 만듭니다. Developer Portal에서 새 애플리케이션을 만들고 Application ID를 기록합니다.
  2. 봇 사용자를 만듭니다. Bot 탭에서 봇 이름과 아바타를 지정합니다. Discord 제공 초대 링크를 쓰려면 Public Bot을 켭니다.
  3. Privileged Gateway Intents를 켭니다. Message Content Intent와 Server Members Intent를 켜고 저장합니다.
  4. 봇 토큰을 발급합니다. 토큰은 한 번만 표시되므로 안전한 비밀 저장소에 보관합니다.
  5. 서버에 초대합니다. bot과 applications.commands scope를 사용하고 필요한 권한을 부여합니다.
  6. Hermes gateway 설정을 실행합니다. 터미널에서 gateway setup을 실행하고 Discord를 선택합니다.
Gateway 설정
hermes gateway setup
hermes gateway status
!

Message Content Intent가 꺼져 있으면 봇이 온라인이어도 메시지 내용을 읽지 못합니다. 개인 서버라면 privileged intent 검증 이슈가 거의 없지만, 봇이 100개 이상 서버에 들어가는 경우 Discord 검증이 필요할 수 있습니다.

3. 설정

설정 목표는 “허가된 사람만”, “필요한 채널에서만”, “불렸을 때만” 반응하게 만드는 것입니다.

권한과 사용자 제한

  • 허용 사용자 목록에는 본인 User ID를 넣습니다.
  • 팀 운영 시에는 역할 기반 허용을 쓰되 역할 범위를 좁힙니다.
  • 봇이 볼 수 있는 채널을 서버 권한에서 제한합니다.
  • 토큰을 재발급했다면 Hermes gateway 설정도 즉시 갱신합니다.

여러 봇이 말싸움하지 않게 하기

두 번째 에이전트는 리뷰 담당으로 두고, 채널의 모든 메시지에 자동 반응하지 않게 설정합니다. 구현 봇이 리뷰어를 멘션할 때만 리뷰어가 끼어드는 구조가 안정적입니다.

리뷰어 봇 설정 방향
discord:
  require_mention: true
  thread_require_mention: true
  history_backfill: true

이종 브리지와 ACP는 고급 옵션

영상에서 다룬 “Codex 구현 + Claude 리뷰” 같은 이종 조합은 매력적이지만, 현재 Hermes 버전에서 해당 provider와 adapter가 실제로 제공되는지 먼저 확인해야 합니다. 공식 provider 목록과 hermes --help, hermes model에 나타나는 항목을 기준으로만 운영에 반영합니다.

!

Claude 계열은 구독 포함량과 API 키 과금 조건이 구분됩니다. 외부 Claude를 붙이는 구성은 비용과 인증 제약을 먼저 확인한 뒤 테스트 채널에서만 검증하세요.

4. 실행

운영 모델은 단순합니다. 사람은 작업 목표를 던지고, 구현 봇은 계획과 패치를 만들며, 리뷰 봇은 근거를 들어 검증합니다.

채널은 프로젝트

프로젝트별로 채널을 분리합니다. 제품, 운영, 리서치, 영상 제작처럼 맥락이 다른 일을 한 채널에 섞지 않습니다.

스레드는 작업

새 작업은 새 스레드에서 시작합니다. 완료 후에는 요약을 남기고 보관하거나 삭제합니다.

상호 리뷰 작업 프롬프트

협업 요청 예시
이 작업을 구현해줘. 먼저 계획을 세운 뒤 리뷰어를 멘션해서 검토를 받아.
리뷰 피드백은 항목별로 동의, 부분 동의, 비동의로 나누고,
쟁점이 정리된 뒤에만 구현을 시작해줘.
완료 후 변경 요약, 검증 결과, 남은 위험을 보고해줘.

비판적 수용 규칙

리뷰어가 항상 맞는 것은 아닙니다. 구현 봇에게 피드백을 그대로 받아들이지 말고 목표, 제약, UX, 유지보수 리스크 기준으로 검토하게 합니다.

리뷰 처리 규칙
리뷰어 피드백을 항목별로 나눠라.
각 항목은 동의, 부분 동의, 비동의로 판단하라.
비동의 항목은 근거를 쓰고 다시 리뷰어와 합의하라.
합의된 항목만 구현 계획에 반영하라.

5. 검증

멀티 에이전트는 “봇이 답한다”만으로 충분하지 않습니다. 권한, 멘션, 스레드, 컨텍스트 분리가 같이 확인되어야 합니다.

  1. 단일 봇 응답 확인허용된 사용자로 메시지를 보내고 응답을 확인합니다.
  2. 비허용 사용자 차단 확인허용 목록 밖 계정 또는 역할이 무시되는지 확인합니다.
  3. 스레드 응답 확인새 스레드에서 봇이 읽고 답하는지 확인합니다.
  4. 리뷰어 멘션 확인리뷰어가 멘션될 때만 반응하고 일반 메시지에는 끼어들지 않는지 확인합니다.
  5. 컨텍스트 점검장기 작업 전후로 세션 크기와 요약 상태를 확인합니다.

6. 장애 대응

증상원인대응
봇이 온라인인데 답하지 않음Message Content Intent 꺼짐Developer Portal에서 intent를 켜고 gateway를 재시작합니다.
Disallowed Intents 오류코드가 요청한 intent와 Portal 설정 불일치필요 intent를 켜거나 Hermes 설정을 줄인 뒤 재시작합니다.
403 Forbidden채널 또는 봇 역할 권한 부족View Channel, Send Messages, Read Message History 권한을 확인합니다.
User not allowed허용 사용자 목록 누락본인 User ID 또는 역할 ID를 허용 목록에 넣고 gateway를 재시작합니다.
여러 봇이 동시에 답함멘션 요구 설정 누락리뷰어 봇에 require_mention과 thread_require_mention을 적용합니다.
작업이 느려짐스레드 컨텍스트 비대화요약 문서를 만들고 새 스레드나 새 세션으로 이어갑니다.

7. 복구

Discord 운영 복구는 토큰, 권한, 컨텍스트, 작업 산출물을 따로 다루면 빠릅니다.

  1. 토큰 유출 또는 의심Discord Developer Portal에서 토큰을 재발급하고 기존 토큰을 무효화합니다.
  2. 봇 폭주gateway를 중지하고, 봇 권한 또는 서버 초대를 임시 제거합니다.
  3. 설정 복구allowed users, roles, channels, mention 요구 설정을 최소값으로 되돌립니다.
  4. 컨텍스트 복구현재 스레드의 목표, 결정, 남은 작업을 요약한 뒤 새 스레드에서 이어갑니다.
  5. 작업물 복구코드나 문서 변경은 Git 히스토리, 백업 저장소, Hermes 세션 요약을 기준으로 되돌립니다.
i

세션을 삭제하기 전에 항상 요약을 먼저 남기세요. 멀티 에이전트 운영에서 가장 비싼 손실은 토큰보다 결정 맥락입니다.

8. 수동 검증 체크리스트

  • Discord Developer Portal 링크가 새 탭으로 열리고 rel="noopener noreferrer"를 포함한다.
  • Message Content Intent와 Server Members Intent 안내가 공식 문서 기준과 맞다.
  • 봇 토큰, User ID 예시가 실제 비밀값이나 임시 토큰을 포함하지 않는다.
  • 리뷰어 봇은 멘션될 때만 반응하도록 설명되어 있다.
  • 키보드만으로 상단 내비게이션, 목차, 복사 버튼, 링크를 사용할 수 있다.
  • 모바일에서 표와 코드 블록이 가로 스크롤로 읽히고 본문이 겹치지 않는다.
  • ACP와 이종 브리지 설명은 현재 버전 확인이 필요한 고급 옵션으로 표시되어 있다.
  • 깨진 내부 앵커와 죽은 로컬 링크가 없다.
